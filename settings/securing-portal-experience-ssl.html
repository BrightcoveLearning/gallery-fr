Sécurisation d'une expérience de portail avec SSL

{{ page.description }}

Secure Sockets Layer (SSL) est la technologie de sécurité standard permettant d'établir un lien chiffré entre un serveur Web et un navigateur. Ce lien garantit que toutes les données transmises entre le serveur Web et les navigateurs restent privées et intégrales. SSL est une norme de l'industrie et est utilisé par des millions de sites Web pour protéger leurs transactions en ligne avec leurs clients.

Note : Par défaut, tous les domaines fournis par Brightcove prennent en charge SSL. Si votre site vidéo utilise l'URL attribuée par défaut ou un domaine fourni par Brightcove (brightcovegallery.com ou gallery.video), il est également possible d'accéder au site en préfixant l'URL par https. Il existe également une option pour appliquer SSL sur ce domaine (recommandé) pour activer une redirection à partir de la version non-SSL du site si une visionneuse y va. Aucun certificat SSL ou configuration n'est requis. Cette rubrique couvre l'utilisation de vos propres certificats SSL avec un domaine personnalisé.

Remarques quant à l'utilisation du SSL avec Gallery

  • Les éditeurs sont responsables du certificat SSL
  • Les certificats autosignés ne sont pas acceptés.
  • La clé privée ne peut pas être cryptée avec un mot de passe.
  • La clé privée doit commencer et se terminer par ‘—BEGIN RSA PRIVATE KEY—’ et ‘—END RSA PRIVATE KEY—’
  • La longueur maximale de clé de certificat SSL prise en charge par Gallery est de 2048 bits
  • Brightcove exige que le certificat soit au format PEM
  • Brightcove recommande que les certificats SSL utilisés dans les sites Gallery ne soient utilisés nulle part ailleurs
  • Les éditeurs doivent utiliser un nom de domaine personnalisé
Note : Lorsque vous travaillez avec des certificats SSL, ne partagez en aucun cas votre clé privée ni ne l'envoyez au support Brightcove.

Configuration d'une expérience de portail pour SSL

Ces étapes supposent que vous avez acheté un certificat SSL. Votre site vidéo doit également être configuré pour utiliser un domaine personnalisé. Pour plus d'informations sur la configuration d'un site pour utiliser un domaine personnalisé, consultez Configuration de domaines personnalisés et SSL.

Upload du certificat SSL

  1. Ouvrez le module Gallery.
  2. Cliquez sur le lien Settings.
  3. Dans le volet de navigation de gauche, cliquez sur Custom Domains & SSL.
  4. Modifiez le domaine personnalisé pour lequel SSL sera activé.
  5. Cochez Use SSL.
  6. Coller dans votre certificat de serveur. Supprimez les lignes vides au début ou à la fin de la clé. Brightcove exige que le certificat soit au format PEM.
  7. Collez votre clé privée de serveur. Supprimez les lignes vides au début ou à la fin de la clé. Brightcove exige que le certificat soit au format PEM.
  8. Coller dans votre chaîne de certificats d'autorité de certification. Si votre certificat a été signé par une autorité de certification racine, seule l'autorité de certification racine doit être collée.  Cela ne doit pas inclure votre propre certificat. La commande de certificat est certifiée immédiate. > tout autre intermédiaire> racine (le certificat racine est facultatif).
    Supprimez les lignes vides au début ou à la fin de la clé. Brightcove exige que le certificat soit au format PEM.
  9. Cliquez sur Enregistrer. Les détails du site seront affichés.
  10. Cliquez sur Save. Le certificat sera validé et s'il y a des erreurs, elles seront affichées.

L'état SSL affiche Mise à jour de la configuration pendant que les informations SSL sont mises à jour et propagées. Cliquez sur Actualiser la liste pour mettre à jour l'état SSL.

Configuration de l'enregistrement CNAME

Une fois SSL configuré, l'état SSL sera activé. Les informations CNAME seront mises à jour avec une URL CloudFront. L'URL CloudFront ne changera que si le domaine est supprimé.

Notez que la navigation directe vers une URL CloudFront renverra une erreur « page introuvable » (404). Une curl commande peut être utilisée pour vérifier la configuration du domaine. Pour cet exemple :

curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

L'instruction curl doit renvoyer le contenu de la page d'accueil du site.

L'enregistrement CNAME de votre domaine personnalisé doit être redirigé vers l'URL CloudFront affichée sur la page Domaines personnalisés et SSL.

Note : Si vous utilisez des en-têtes et pieds de page personnalisés, vérifiez que toutes les URL sont également sécurisées.

Mise à jour et remplacement des certificats SSL

Si un certificat SSL doit être mis à jour ou remplacé, par exemple, le certificat SSL peut être défini pour expirer, procédez comme suit pour remplacer un certificat SSL existant.

  1. Ouvrez le module Gallery.
  2. Cliquez sur le lien Settings.
  3. Dans le volet de navigation de gauche, cliquez sur Custom Domains & SSL.
  4. Localisez le domaine personnalisé à l'aide du certificat SSL, puis cliquez sur l'icône de modification ().
  5. Cliquez sur Charger un nouveau certificat.
  6. Coller dans le nouveau certificat de serveur, clé privée de serveur et chaîne de certificats d'autorité de certification. Supprimez les lignes vides au début ou à la fin des touches.
  7. Cliquez sur Save.

S'il y a des erreurs avec les clés mises à jour, les anciennes valeurs seront conservées.

FAQ

  1. Un site Gallery avec un domaine personnalisé et SSL peut-il être accessible uniquement via HTTPS ? Oui, en fait, c'est la seule façon d'autoriser l'accès au site. Si vous essayez d'accéder à un site sur le lien HTTP, vous serez automatiquement redirigé vers le site HTTPS. Ceci est fait en forçant les redirections HTTPS dans CloudFront.
