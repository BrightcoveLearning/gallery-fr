Configuration des domaines personnalisés et SSL

{{ page.description }}
 

Lorsque vous créez une expérience de portail pour la première fois, un domaine aléatoire lui est attribué au format de <site-slug>.bcvp0rtal.com . Par exemple, http://site-148393.bcvp0rtal.com . Lorsque le site est publié, les visiteurs sont en mesure d'y accéder par ce domaine. Gallery permet de publier des sites vidéo dans un domaine fourni par Brightcove ou sur votre propre domaine personnalisé.

Les expériences de portail Galerie peuvent également être attribuées à plusieurs URL permettant aux éditeurs d'utiliser le même site pour différentes fonctions sans avoir à dupliquer le site. Par exemple, si vous avez un site vidéo que vous souhaitez utiliser dans une campagne et comme portail vidéo principal, vous pouvez marquer différentes URL et les affecter toutes au même site. Aucune duplication de site n'est nécessaire et vous pouvez facilement supprimer les URL lorsqu'elles ne sont plus nécessaires.

Note : Lorsque vous travaillez avec des certificats SSL, ne partagez en aucun cas votre clé privée ni ne l'envoyez au support Brightcove.

Utilisation d'un domaine fourni par Brightcove

Les domaines fournis par Brightcove vous permettent de créer rapidement et en toute simplicité un domaine pour votre site sans requérir l'assistance de votre service informatique pour créer un CNAME personnalisé ou enregistrer un nouveau nom de domaine auprès d'un bureau d'enregistrement des noms de domaines. Ils sont disponibles « hors de la boîte » et peuvent être mis en œuvre en un clic de bouton. Brightcove fournit trois domaines :

  • brightcovegallery.com
  • gallery.video
  • bcvp0rtal.com (affecté aux sites par défaut)

Tous prennent en charge l'accès SSL. Pour accéder au site par une connexion SSL, il vous suffit d'ajouter https à l'URL. Par exemple, https://marquee.brightcovegallery.com

Utilisation d'un domaine personnalisé (enregistré par l'éditeur)

Les sites vidéo peuvent également être hébergés sur votre propre domaine personnalisé que vous enregistrez. Cela vous permet de nommer l'URL de façon cohérente avec la dénomination de votre site internet ou de votre URL actuelle. Pour ce faire, vous devrez d'abord enregistrer le nom de domaine puis configurer un enregistrement CNAME personnalisé auprès de votre hébergeur. Pour obtenir des informations sur la configuration d'un enregistrement CNAME, consultez la section ci-dessous.

Création d'un domaine personnalisé

Pour créer un domaine personnalisé, procédez comme suit :

  1. Ouvrez le module Gallery.
  2. Cliquez sur le lien Paramètres de la page d'accueil de la Galerie.
  3. Dans la navigation de gauche, cliquez sur le DOMAINES PERSONNALISÉS & SSL relier.
  4. Cliquez sur Create Custom Domain.
  5. Sélectionnez une option de domaine à utiliser :
    • brightcovegallery.com - Le domaine du site aura le format <your domain>.brightcovegallery.com.
    • gallery.video - Le domaine du site aura le format <your domain>.gallery.video .
    • Personnalisé : sélectionnez cette option pour héberger le site sur votre propre domaine personnalisé. Lorsque vous sélectionnez cette option, vous devez également configurer un enregistrement CNAME auprès de votre hébergeur.
  6. Saisissez le nom de domaine. Un message s'affiche, indiquant si le domaine est valide ou non. Si le nom de domaine n'est pas valide, entrez un nouveau nom de domaine.
  7. Si vous publiez le site à l'aide d'un domaine fourni par Brightcove, il existe une option pour appliquer SSL sur ce domaine. Cochez Appliquer SSL sur ce domaine (recommandé) pour activer une redirection à partir de la version non-SSL du site si une visionneuse y va.
  8. Si vous publiez le site sur votre propre domaine personnalisé, vous avez la possibilité de sécuriser le site via SSL. Cochez la case Use SSL pour sécuriser le site à via SSL. Pour plus d'informations sur l'utilisation de SSL, voir Sécurisation d'une expérience de portail avec SSL.
  9. Cochez Activer la redirection pour rediriger cette URL vers une autre URL. Généralement, cela ne sera vérifié qu'après que ce domaine a été utilisé pendant un certain temps. Lorsque cette option est sélectionnée, entrez l'URL de redirection vers laquelle cette URL existante doit être redirigée. Cochez Utiliser une redirection permanente (code d'état HTTP 301) pour indiquer que la redirection est permanente. Lorsqu'une araignée de moteur de recherche trouve un code d'état 301 dans l'en-tête de réponse d'une page Web, elle comprend que cette page Web n'existe plus et remplace l'URL indexée par la nouvelle. Cette option conservera le trafic et le classement de votre ancienne page Web.
  10. Cliquez sur Save. Une liste des domaines configurés s'affiche.
    Note : Lorsque vous utilisez un domaine fourni par Brightcove, l'état SSL est toujours fourni. Cela indique qu'en ajoutant https :// au début de l'URL du site, la transmission des données entre votre navigateur et le site Gallery sera sécurisée.
  11. Créez des domaines supplémentaires si nécessaire.

Pour modifier ou supprimer un domaine, cliquez sur le bouton Modifier () ou supprimer ().

Une fois les domaines personnalisés créés, vous pouvez leur affecter à un site en fonction des propriétés du site. Pour plus d'informations, voir Affectation d'un domaine personnalisé à une expérience de portail.

Configuration des enregistrements CNAME avec votre fournisseur d'hébergement

Note : Les enregistrements CNAME ne doivent être configurés que lorsque des domaines personnalisés seront attribués à un site. IL N'EST PAS nécessaire de créer des enregistrements CNAME pour les sites utilisant des domaines fournis par Brightcove.

Les étapes de configuration d'un enregistrement CNAME (Canonical Name) diffèrent en fonction de votre fournisseur d'hébergement. Vous devez configurer un enregistrement CNAME pour chaque domaine personnalisé que vous utiliserez. Si votre site est configuré pour utiliser SSL, l'enregistrement CNAME doit pointer vers l'URL CloudFront. Voir Sécurisation d'une expérience de portail avec SSL pour plus d'informations. Notez que les domaines personnalisés doivent inclure un sous-domaine afin d'être configuré en tant que CNAME ex : video doit être placé avant brightcovelearning.com afin que je puisse configurer mon CNAME.

Les valeurs CNAME sont affichées dans les paramètres Custom Domains et SSL.

Dans cet exemple, pour le nom de domaine video.brightcovelearning.com , il convient d'ajouter un enregistrement CNAME et un hôte de video et pointez-le vers d3mkhzp6k6x3hl.cloudfront.net.

Notez que la navigation directe vers une URL Cloudfront renverra une erreur « page not found » (404). Une curl commande peut être utilisée pour vérifier la configuration du domaine. Pour cet exemple :

curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

L'instruction curl doit renvoyer le contenu de la page d'accueil du site.

Vous devrez peut-être contacter votre administrateur réseau ou votre fournisseur d'hébergement Web pour obtenir de l'aide sur cette étape. Il faut parfois patienter 48 heures avant que le changement de DNS ne soit fonctionnel.

Sécurisation des sites via SSL

Il existe deux options pour sécuriser une expérience de portail avec SSL :

  1. Pour les sites publiés à l'aide de l'identifiant de site affecté (par ex. <site-slug>.bcvp0rtal.com ) ou qui utilisent un domaine fourni par Brightcove, aucune configuration n'est nécessaire pour prendre en charge les connexions https. L'ajout de https :// au début de l'URL permettra de sécuriser la transmission des données entre votre navigateur et le site Gallery. Vous pouvez également sélectionner Appliquer SSL sur ce domaine (recommandé) pour activer une redirection à partir de la version non-SSL du site si une visionneuse y va.
  2. Pour les sites hébergés à l'aide de votre domaine propre domaine personnalisé, cochez l'option Use SSL lorsque vous saisissez votre nom de domaine. Vous serez invité à saisir les informations de votre certificat. Pour plus d'informations, consultez Sécurisation d'une expérience de portail avec SSL.
