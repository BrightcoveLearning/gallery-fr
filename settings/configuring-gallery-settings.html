    Configuration des paramètres de Gallery

    Cette rubrique contient une présentation des paramètres de Gallery configurables.
     

    Pour configurer les paramètres de la Galerie, ouvrez le module Galerie et cliquez sur le lien Paramètres en haut de la page. Les paramètres suivants peuvent être configurés :

    • Joueurs
    • YouTube
    • Domaines personnalisés et SSL
    • Profils de contrôle d'accès
    • HTML personnalisé

    Sélection d'un lecteur Video Cloud par défaut

    Gallery doit être configuré avec un lecteur vidéo par défaut pour l'ensemble du compte qui sera utilisé sur vos sites Gallery. Les joueurs des sites Gallery individuels peuvent également être configurés dans le cadre des paramètres VIDÉO ET LECTURE, ce qui permet de remplacer le lecteur par défaut pour l'ensemble du compte. Pour plus d'informations, voir Personnalisation des paramètres de formulaire Player et Lead for a Portal Experience.

    En tant que meilleure pratique, nous vous recommandons de créer et de styliser des joueurs dédiés pour vos sites Galerie. Si vous n'avez pas créé de lecteur dédié pour votre site Galerie, vous devriez commencer par le faire. Pour plus d'informations sur la création de joueurs, voir Mise en route avec le module Joueurs.

    Pour configurer Gallery avec un lecteur vidéo par défaut, procédez comme suit.

    1. Cliquez sur Joueurs dans la navigation de gauche.
    2. Sélectionnez le lecteur par défaut de votre compte dans la liste des joueurs, puis cliquez sur Enregistrer.
    Remarque : Un lecteur par défaut doit être sélectionné, sinon Portal Experiences échoue à publier.

    Les lecteurs compatibles avec le public sont indiqués par l'icône Audience (). Ces lecteurs sont configurés avec le plugin Audience et peuvent suivre les données d'engagement vidéo. Le survol de l'icône affichera les informations de connexion de l'audience.

    Pour plus d'informations sur la création de lecteurs compatibles avec l'audience, reportez-vous à la section Création d'un lecteur compatible Audience.

    Configuration des paramètres YouTube

    Les sites Gallery peuvent contenir des vidéos de votre compte Video Cloud ou d'un compte YouTube. Avant d'ajouter des vidéos YouTube à un site Gallery, les paramètres YouTube doivent être configurés. Pour plus d'informations sur ces paramètres, consultez Configuration des paramètres YouTube.

    Configuration de domaines personnalisés et SSL

    Lorsque vous créez un site Gallery, un domaine aléatoire au format <site-slug>.bcovp0rtal.com lui est affecté. Gallery permet de publier des sites Gallery dans un domaine fourni par Brightcove ou en utilisant votre propre domaine personnalisé. Les sites peuvent également être sécurisés à l'aide de SSL. Pour plus d'informations sur la configuration des domaines personnalisés et SSL, reportez-vous à la section Configuration des domaines personnalisés et SSL.

    Configuration des profils de contrôle d'accès

    Il est possible de créer des profils de contrôle d'accès pour contrôler l'accès aux sites Gallery. L'accès à un site peut être limité par :

    • Adresse IP - L' accès est limité en fonction de l'adresse IP du spectateur
    • Informations d'identification SSO - Un nom d'utilisateur dont un mot de passe est requis pour l'accès
    • Code d'accès - L'utilisateur doit entrer un code d'accès avant que le site ne s'affiche (utile lors de la création et du test d'un site)

    Pour plus d'informations sur la création de profils de contrôle d'accès, voir Contrôle de l'accès à une expérience de portail.

    Utilisation du HTML en-tête/pied de page personnalisé

    Gallery propose un système d'en-tête et de pied de page personnalisés pour vos sites. Cela permet de personnaliser vos sites Gallery afin qu'ils correspondent plus étroitement à l'apparence de vos sites d'entreprise. Vous pouvez également définir un en-tête/pied de page personnalisé pour l'ensemble du site. Pour plus d'informations sur la configuration d'un en-tête/pied de page personnalisé pour un site, voir Personnalisation de l'apparence et du comportement d'une expérience de portail.

    Pour configurer Gallery pour qu'il utilise un en-tête ou un pied de page personnalisé, commencez par concevoir l'en-tête/pied de page à utiliser pour vos sites. Il peut contenir n'importe quel code HTML, JavaScript ou CSS valide. Lorsque vous avez terminé, procédez comme suit.

    1. Cliquez sur HTML personnalisé dans la navigation de gauche.
    2. Collez le code HTML de l'en-tête dans le champ Custom Header HTML.
    3. Collez le code HTML du pied de page dans le champ Custom Footer HTML.
    4. Cliquez sur Save pour enregistrer les modifications apportées.

    Remarques sur l'utilisation d'en-têtes et de pieds de page personnalisés

    • Gallery ne valide pas le code HTML fourni pour l'en-tête/pied de page
    • Il n'y a pas de limite de caractères sur l'en-tête et le pied de page HTML.
    • Les fichiers CSS et JavaScript externes sont pris en charge.
    • Le code HTML personnalisé doit suivre les principes de conception réactive afin qu'il évolutie correctement sur tous les appareils
    • Brightcove a vu des conflits CSS lorsque les sites utilisent Bootstrap (Gallery utilise également Bootstrap) ; vérifiez et résolvez les conflits
