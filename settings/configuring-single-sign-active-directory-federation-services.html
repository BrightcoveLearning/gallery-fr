Configuration de l'authentification unique avec les services de fédération Active Directory

{{ page.description }}

Avant de commencer

Effectuez les étapes de configuration initiales suivantes :

  1. Activez RelayState pour la connexion initiée par l'IDP en suivant les instructions ici. L'emplacement varie en fonction de la version d'ADFS en cours d'utilisation.
  2. Redémarrez le service ADFS pour que les modifications soient appliquées.

Ajout d'un profil de contrôle d'accès Galerie

  1. Ouvert Outils administratifs à partir du menu Démarrer de Windows ou du Panneau de configuration, puis ouvrez le Gestion ADFS application.
  2. Ouvrez > Certificats de services dans le panneau de l'explorateur de gauche.
  3. Double-cliquez sur le certificat de signature de jeton que vous souhaitez utiliser.
  4. Cliquez sur l'onglet Détails et cliquez sur Copier dans le fichier...
  5. Cliquez sur Prochain dans l'assistant d'exportation, puis sélectionnez le X.509 codé en Base-64 (.CER) option.
  6. Enregistrez le fichier de certificat dans votre système de fichiers local, puis ouvrez-le dans un éditeur de texte. Il s'agit du certificat qui sera utilisé pour configurer le profil de contrôle d'accès Gallery.
  7. Ouvrez le module Gallery.
  8. Cliquez sur Settings (Paramètres).
  9. Cliquez sur Profils de contrôle d'accès dans la navigation de gauche.
  10. Cliquez sur Create Profile.
  11. Donnez-lui un nom, puis sélectionnez SSO - Nécessite un nom d'utilisateur et un mot de passe pour y accéder.
  12. Dans le Point de terminaison SAML 2.0 (HTTP) , entrez l'URL de la page de connexion initiée par iDP pour votre serveur ADFS. Cela se termine habituellement par IdpInitiatedSignOn.aspx. Par exemple, si le lien de connexion initié par le SP est https://sso-test.brightcove.com/adfs/ls, la page de connexion initiée par l'IDP est https://sso-test.brightcove.com/adfs/ls/IdpInitiatedSignOn.aspx.
  13. Cochez l'option My SSO System is ADFS.
  14. Copiez et collez le certificat que vous avez enregistré à l'étape 6 dans le Certificat X.509 champ.
  15. Cliquez sur Enregistrer pour enregistrer le profil de contrôle d'accès. Le nouveau profil de contrôle d'accès doit apparaître dans la liste des profils.
  16. Dans l'application de gestion AD FS, ouvrez les approbations de partie de confiance.
  17. Cliquez sur Ajouter l'approbation d'une partie de confiance... dans le panneau de droite.
  18. Cliquez sur Début dans l'assistant puis collez le URL de métadonnées qui a été affiché à l'étape 15 dans le Adresse de métadonnées de fédération champ.
  19. Cliquez sur Suivant, puis donnez un nom à votre partie de confiance.
  20. Continuez à cliquer Prochain jusqu'à ce que le Finir étape. Décochez la case Configurer la stratégie d'émission des réclamations pour cette application. Cliquez sur Close (Fermer).

Vous devriez désormais pouvoir associer une expérience de portail au profil de contrôle d'accès créé. Une fois l'expérience publiée, la navigation vers elle doit passer par le flux de travail de connexion ADFS.
