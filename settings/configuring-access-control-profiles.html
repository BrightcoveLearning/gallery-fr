Configuration des profils de contrôle d'accès

{{ page.description }}

Par défaut, lorsqu'une expérience de portail Galerie est publiée, toute personne disposant de l'URL du site pourra accéder au site. Il est possible de créer des profils de contrôle d'accès pour restreindre l'accès à une expérience de portail Galerie. Les profils de contrôle d'accès sont créés dans les paramètres Gallery puis affectés aux sites. Les profils de contrôle d'accès limitent l'accès en fonction des éléments suivants :

  • Adresse IP
  • Sign-on unique
  • Code d'accès

Notez qu'une expérience de portail Galerie ne peut être configurée qu'avec un seul profil de contrôle d'accès. Chaque profil de contrôle d'accès ne peut avoir qu'un seul type de restriction d'accès. Par exemple, il n'est pas possible de créer un profil qui utilise à la fois une adresse IP et un code d'accès. En outre, lorsque la connexion unique est utilisée, un seul fournisseur d'identité peut être configuré par profil.

Pour créer un profil de contrôle d'accès, procédez comme suit :

  1. Ouvrez le module Gallery.
  2. Cliquez sur le lien Paramètres de la page d'accueil de la Galerie.
  3. Dans la navigation de gauche, cliquez sur Profils de contrôle d'accès. Une liste des profils de contrôle d'accès créés s'affiche.
    Note : Vous pouvez voir dans votre compte des profils de contrôle d'accès que vous n'avez pas créés. Si vous aviez configuré des restrictions IP, une authentification unique ou un code d'accès pour les sites avant que Gallery n'implémente les profils de contrôle d'accès, ces paramètres ont été enregistrés en tant que profils de contrôle d'accès lorsque la fonctionnalité a été publiée.
  4. Cliquez sur Create Profile.
  5. Saisissez un nom.
  6. (Facultatif) Configurez un délai d'expiration de session pour définir la durée d'une session utilisateur. Remarques :
    • Ce paramètre s'applique lorsque vous utilisez l'authentification unique et un code d'accès.
    • Il s'agit d'un délai d'inactivité. Chaque fois qu'un serveur Gallery est sollicité (soit par le chargement d'une page sur le site soit via un appel d'API) le minuteur est réinitialisé. Cela permet aux visiteurs de continuer à parcourir le site plus longtemps que le délai d'expiration spécifié. Toutefois, dès que l'inactivité dépasse le délai d'expiration, les visiteurs sont déconnectés. Comme le serveur Gallery n'est pas sollicité pendant le visionnage d'une vidéo, la valeur du délai d'expiration doit être supérieure à la durée de votre vidéo la plus longue.
    • Votre fournisseur de services SSO doit être configuré de façon à forcer la réauthentification ou à appliquer un délai d'expiration inférieur au nôtre. Dans le cas contraire, l'utilisateur sera redirigé vers votre fournisseur de services SSO après l'expiration du délai. Celui-ci renverra une réponse indiquant que l'utilisateur est connecté.
  7. Sélectionnez les options pour sécuriser le site. Une seule option par profil de contrôle d'accès peut être sélectionnée:
    • Adresse IP
    • Sign-on unique
    • Code d'accès
  8. Cliquez sur Save.
  9. Créez des profils de contrôle d'accès supplémentaires si nécessaire.

Pour modifier ou supprimer un profil de contrôle d'accès, cliquez sur le bouton Modifier () ou supprimer () associés au profil de contrôle d'accès. Une fois les profils de contrôle d'accès créés, ils peuvent être affectés à un site dans le cadre des propriétés du site. Pour plus d'informations, consultez Contrôle de l'accès à une expérience de portail.

Configuration de la restriction IP

Si vous créez un profil de contrôle d'accès fondé sur la restriction d'IP, les visiteurs disposant d'une adresse IP en dehors de la plage prédéfinie recevront un message d'erreur lorsqu'ils tenteront d'accéder au site.

Pour créer un profil de contrôle d'accès fondé sur la restriction d'IP, procédez comme suit :

  1. Sélectionnez le Limité par l'adresse IP option.
  2. Saisissez la plage d'adresses IP autorisées à accéder au site. La syntaxe de bloc CIDR est également prise en charge.
  3. Cliquez sur Save.

Configuration de l'authentification unique

Si vous créez un profil de contrôle d'accès fondé sur l'authentification unique, les visiteurs du site seront invités à se connecter avec des identifiants corrects avant de pouvoir accéder au site. Gallery prend en charge la norme SAML 2.0 (Security Assertion Markup Language 2.0) pour l'échange de données d'authentification et d'autorisation entre domaines de sécurité.

Gallery prend en charge la plupart des principaux fournisseurs d'identité, notamment :

  • Ping
  • OneLogin
  • ADFS 2.0/3.0
  • Google G Suite
  • Shibboleth
  • Okta
  • Salesforce.com

D'autres fournisseurs utilisant la norme SAML 2.0 fonctionneront également. La galerie utilise le passeport-saml une bibliothèque. Gallery ne prend pas en charge les fournisseurs d'identité qui ne fonctionnent pas avec cette bibliothèque.

Note : Gallery nécessite SAML 2.0 pour l'authentification par authentification unique. SAML 1.1 n'est pas pris en charge.
Note : Nous ne prenons en charge que les caractères à un octet.
Note : Gallery ne prend en charge que les flux SAML initiés par le fournisseur d'identité (IDP). Les flux initiés par un fournisseur de services ne sont pas pris en charge.

Votre fournisseur SSO aura besoin d'une URL de service client d'assertion (URL ACS). L'URL ACS est répertoriée dans la section SSO du tableau Profils de contrôle d'accès.

L'URL ACS est par défaut https://auth.bcvp0rtal.com/login/callback. Vous pouvez également remplacer l'URL ACS par votre propre URL. Ces deux options permettent de configurer des fournisseurs de services SSO plus stricts. Pour les sites existants, l'URL http(s)://<site-root-url>/login/callback fonctionnera dans tous les cas, mais Brightcove recommande aux éditeurs d'utiliser l'URL https://auth.bcvp0rtal.com/login/callback. Ce point de terminaison prend en charge le transfert des données SAML vers le site correct pour tous les sites qui utilisent un chemin d'accès sur un domaine.

Note : Lorsque vous utilisez un flux de travail SSO initié par ID, l'URL ACS spécifiée dans votre logiciel SSO doit pointer vers un site Gallery configuré avec ce profil de contrôle d'accès. Dans ce cas, l' https://auth.bcvp0rtal.com/login/callback URL ne doit pas être utilisée.

Pour configurer un profil de contrôle d'accès pour utiliser l'authentification SSO, procédez comme suit :

  1. Sélectionnez le authentification unique option.
  2. Si vous disposez d'un fichier d'exportation de vos métadonnées SAML 2.0, cliquez sur Sélectionner un fichier pour parcourir et sélectionner le fichier de métadonnées. Cela préremplira le Point de terminaison SAML 2.0 et le Certificat X.509 des champs.
  3. Entrez une valeur pour le point de terminaison SAML 2.0 (HTTP). Cette valeur doit être disponible auprès de votre fournisseur SAML.
  4. Saisissez la valeur de X.509 Certificate. Cette valeur doit être disponible auprès de votre fournisseur SAML.
  5. (Facultatif) Saisissez l'URL ou l'ID SAML Issuer / Entity ID Override. Saisissez une valeur ici pour passer outre l'émetteur SAML.
  6. (Facultatif) Saisissez l'URL ACS URL Override. Saisissez une valeur ici pour passer outre l'URL ACS.
  7. (Facultatif) Entrez le paramètre SAML AuthNContext Remplacer. Entrez une valeur ici pour remplacer éventuellement le SAML AuthnContext.
  8. (Facultatif) Choisissez de désactiver le AuthnContext par défaut. Par défaut, pour les profils SSO, Gallery enverra PasswordProtectedTransport en tant que contexte d'authentification SAML. Parfois, cela n'est pas pris en charge par les fournisseurs d'identité. Lorsque cette option est cochée, la Galerie n'enverra pas la demande de contexte d'authentification.
  9. (Facultatif) Saisissez tout attribut SAML à transférer à votre JavaScript externe.
  10. (Facultatif) Cochez la case Expose all attributs pour exposer tous les attributs de SAML à utiliser dans un code JavaScript externe.
  11. Cliquez sur Save.

Configuration d'un code d'accès

Lorsqu'un site a un code d'accès appliqué, les utilisateurs sont invités à fournir le code d'accès correct lorsque l'URL du site accède.

Si un code d'accès incorrect est entré, le spectateur recevra une erreur. Les codes d'accès sont un bon moyen pour limiter l'accès à un site lorsque celui-ci est en cours de développement. En revanche, les codes d'accès n'offrent pas de sécurité ou de système d'authentification au niveau du site.

Pour configurer un profil de contrôle d'accès fondé sur un code d'accès, procédez comme suit :

  1. Sélectionnez l'option Require Access Code .
  2. Entrez le code d'accès du site.
  3. Saisissez l' étiquette du code d'accès. L' étiquette du code d'accès apparaîtra au-dessus de laquelle l'utilisateur saisira son code d'accès.
  4. (Facultatif) Stylisez la page de contrôle d'accès.
  5. Cliquez sur Save.

Styliser la page de contrôle d'accès

La page de contrôle d'accès peut être stylisée de manière à correspondre plus étroitement à l'apparence de votre site global. Gallery propose trois options de style :

  • Par défaut
  • Image et bordure
  • Personnalisé

Style par défaut

Avec l'option de style par défaut, un logo Galerie, l'étiquette du code d'accès et un champ Code d'accès s'affichent toujours.

Style d'image et de bordure

L'option Image et bordure permet de personnaliser la page de contrôle d'accès. Un exemple peut être vu ci-dessous.

Pour personnaliser le style de l'image et de la bordure, procédez comme suit :

  1. Sélectionnez Image et bordure.
  2. Faites glisser et déposez, parcourez ou ajoutez un lien vers l'image à afficher sur la page de contrôle d'accès.
  3. Saisissez une URL de clics. L' URL de clickthrough fera de l'image un lien cliquable.
  4. Cliquez sur le sélecteur de couleurs pour sélectionner une couleur de bordure d'en-tête.
  5. Cliquez sur Save.

Style personnalisé

L'option de style personnalisé permet des personnalisations plus avancées de la page de contrôle d'accès. Un exemple peut être vu ci-dessous.

Pour utiliser le Personnalisé option, procédez comme suit :

  1. Sélectionnez Personnalisé.
  2. Entrez un code HTML personnalisé pour l'en-tête et le pied de page.
  3. Cliquez sur Enregistrer
