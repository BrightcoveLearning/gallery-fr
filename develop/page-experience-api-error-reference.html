Référence des erreurs de l'API d'expérience en page

Cette rubrique fournit une référence d'erreur pour l'API In-Page Experience (l'API de plate-forme).

Erreurs de l'API d'expérience en page

Référence des erreurs de l'API d'expérience en page
Erreur Exemples Description
400 BAD_REQUEST 
{
  "status": 400,
  "code": "BAD_REQUEST",
  "message": "No template specified"
}
{
  "status": 400,
  "code": "BAD_REQUEST",
  "message": "Specified template does not exist"
}
{
  "status": 400,
  "code": "BAD_REQUEST",
  "details": [
    "Path `name` is required."
  ]
}
 Généralement un problème avec le corps de la requête, comme omettre le champ template ou un autre champ obligatoire, ou spécifier un modèle qui n'existe pas
401 UNAUTHORIZED 
[
  {
    "error_code": "UNAUTHORIZED",
    "message": "Permission denied."
  }
]
 Soit aucun jeton d'accès n'a été envoyé avec la demande, soit les informations d'identification du client utilisées pour créer le jeton d'accès n'avaient pas les autorisations correctes pour la demande
404 NOT_FOUND 
{
  "status": 404,
  "code": "NOT_FOUND",
  "message": "Experience 5bc2624e0f6f340011b not found"
}
 L'expérience que vous essayez de mettre à jour n'existe pas. Il peut avoir été supprimé, ou l'identifiant d'expérience dans la demande peut être incorrect.
