Brève introduction : API REST (In-Page Experience Platform)

Le démarrage rapide fournit des exemples de demandes d'API pour la gestion des expériences en page et fournit un exemple d'interface pour les utiliser.

Introduction

L'API In-Page Experience vous permet de gérer les expériences en page par programme, ce qui la rend particulièrement utile pour les intégrations CMS, pour permettre aux utilisateurs de créer des expériences de gestion à partir de leur propre CMS au lieu d'aller à Gallery in Studio.

Ce démarrage rapide fournit un exemple d'implémentation qui permet aux utilisateurs de :

  1. Obtenir tous les modèles d'expérience en page pour un compte
  2. Créer une expérience de base à partir du modèle sélectionné
  3. Sélectionnez une liste de lecture ou une liste de vidéos à ajouter à l'expérience
  4. Mettre à jour l'expérience avec les vidéos/playlist sélectionnés
  5. Publier l'expérience

Voici une vue de haut niveau de la logique de l'application.

Code proxy
