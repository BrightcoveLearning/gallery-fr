Introduction

Il existe deux API associées aux expériences en page, mais elles sont utilisées à des fins très différentes :

API Expériences en page (REST)

L'API In-Page Experiences est une API REST qui vous permet de gérer les expériences en page par programme. Il est principalement destiné à l'intégration de la plateforme Video Cloud à votre application CMS.

Consultez également la référence complète de l'API pour plus de détails sur les demandes disponibles.

API client Expériences en page

L'API Client In-Page Experiences est une bibliothèque JavaScript qui vous aide à gérer le comportement de l'expérience en page au moment de l'exécution, de la même manière que l'API Brightcove Player est utilisée pour contrôler le comportement du lecteur. L'API est disponible sur n'importe quelle page exécutant une ou plusieurs expériences intégrées Brightcove. Voir la présentation : In-Page Experience Client API pour plus d'informations.

Le reste de cette présentation traite de l'API REST pour gérer vos expériences en page sur le backend.

Pour obtenir une présentation générale des expériences dans la page, reportez-vous à Généralités : Création, modification et publication d'expériences dans la page.

Les sections qui suivent fournissent une vue d'ensemble des 4 groupes de requêtes API REST :

  • Expériences
  • Modèles
  • Thèmes
  • Polices

Expériences

Les demandes d'expériences se répartissent en trois sous-catégories :

Gérer les expériences
Ces demandes vous permettent d'obtenir une liste de toutes les expériences pour le compte. Vous pouvez également créer une nouvelle expérience et obtenir, mettre à jour ou supprimer une expérience spécifique. Enfin, vous pouvez dupliquer une expérience pour créer un point de départ pour une nouvelle expérience.
Publiser/Dépublier des expériences
Ces demandes à tous pour obtenir une liste de tous les modèles pour le compte ou un modèle particulier.
Gérer l'état des expériences en direct
Ces deux demandes vous permettent soit d'obtenir le statut actuel d'une expérience en direct, soit de modifier cet état (de preEvent à live , par exemple).

Modèles

Les modèles contrôlent l'aspect général d'une expérience. Les modèles personnalisés ne peuvent pas être créés, mais il existe plusieurs modèles standard, et d'autres peuvent être ajoutés au fil du temps. L'API vous permet d'obtenir une liste de modèles disponibles pour votre compte ou d'extraire un modèle spécifique.

Thèmes

Les thèmes vous permettent de personnaliser un modèle pour créer l'apparence que vous voulez pour votre expérience. L'API fournit des méthodes pour gérer entièrement vos thèmes, en les créant, en les obtenant, en les mettant à jour ou en les supprimant.

Polices

Vous pouvez utiliser vos propres polices. Ces méthodes vous permettent d'ajouter des polices, de les mettre à jour, de les obtenir ou de les supprimer. Ce sont des polices Web, chargées via CSS.
