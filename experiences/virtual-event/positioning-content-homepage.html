Positionnement du contenu sur la page d'accueil

{{ page.description }}

Plusieurs objets du fichier de configuration ont des paramètres qui vous permettent de spécifier où l'objet doit apparaître sur la page. Par exemple, le contenu statique, les conférenciers, les pistes et les sponsors ont tous la possibilité d'être repositionnés sur la page. Les objets qui peuvent être positionnés auront les attributs suivants dans le fichier de configuration :

  • insert_target- Un sélecteur CSS (classe ou ID simple) qui sera utilisé comme point de départ du rendu de l'objet donné
  • insert_mode- Comment le contenu est inséré
    • after
    • before
    • append
    • prepend

Détermination de la cible

Le insert_target paramètre attend un sélecteur CSS (en utilisant un format simple de classe ou d'ID). L'élément correspondant par ce sélecteur sera utilisé comme point de départ pour le rendu de l'objet donné. La meilleure façon de déterminer le sélecteur CSS ou l'ID à utiliser est de parcourir la page d'accueil de l'événement virtuel, puis d'utiliser les outils de développement du navigateur pour inspecter les éléments de la page.

Vous remarquerez que les principales sections de la page d'accueil sont organisées à l'aide de <div> balises.

Choix du mode d'insertion

Après avoir déterminé où insérer l'objet dans la hiérarchie, l' insert_mode attribut détermine la façon dont l'objet est inséré dans la hiérarchie.

  • after- Place l'objet après la valeur spécifiée div au même niveau
  • before- Place l'objet avant l'objet spécifié div au même niveau
  • append- Place l'objet à l'intérieur du champ spécifié div en tant que dernier enfant
  • prepend- Place l'objet à l'intérieur du champ spécifié div en tant que premier enfant

Exemple de positionnement d'objet

Les exemples de code suivants montrent comment utiliser un objet de contenu statique pour insérer un titre au-dessus de la collection secondaire.


{
    "insert_target": ".bc-on-demand-section",
    "insert_mode": "prepend",
    "content": "<h2 class='bc-on-demand-section-title'>ON-DEMAND SESIONS</h2>",
    "require_roles": [],
    "omit_roles": []
}

En utilisant les outils du navigateur, vous pouvez voir le h2 a été inséré comme le premier élément à l'intérieur du bc-on-demand-section div.
