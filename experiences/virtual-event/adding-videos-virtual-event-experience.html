Ajout de vidéos à une expérience d'événement virtuel

{{ page.description }}

À l'aide de l'éditeur de collection Gallery, vous pouvez sélectionner les vidéos à afficher sur une expérience d'événement virtuel. Comme d'autres expériences de type portatif, les vidéos sont organisées en collections.

Pour ajouter des vidéos à une expérience d'événement virtuel, cliquez sur VIDÉOS dans la navigation de gauche.

Organisation de la collection vidéo

L'ordre des collections contrôlera leur apparence sur la page d'accueil de l'événement virtuel.

Note : Pour plus d'informations sur la création de collections et l'ajout de vidéos à une collection, voir Ajout de vidéos à une expérience de portail.

Le modèle d'événement virtuel comporte quatre zones principales où les vidéos sont affichées comme indiqué dans le site exemple :

  1. Collection en vedette - Généralement utilisée pour présenter le contenu live ou simulive qui sera affiché tout au long de l'événement. Cela devrait idéalement être commandé en fonction de son temps de mise en service du plus tôt au dernier, pour une expérience optimale.
  2. Collection principale : peut inclure n'importe quoi, mais il est généralement utilisé pour présenter des cassures de VOD spécifiques ou du contenu sur lequel vous souhaitez attirer l'attention.
  3. Collection secondaire : peut inclure n'importe quoi, mais est généralement utilisé pour présenter des cassures de VOD ou du contenu spécifiques auxquels vous souhaitez attirer l'attention.
  4. Pistes : généralement utilisées pour le contenu VOD
Note : Les collections Featured, Primary et Secondary prennent en charge un maximum de 24 vidéos.
Note : Pour insérer un titre au-dessus de la collection primaire/secondaire, ajoutez un objet de contenu statique au fichier de configuration. Pour plus d'informations, voir Utilisation du contenu statique.

Lors de la commande de collections dans la Galerie, les collections doivent être commandées de la manière suivante :

  • Collection 1 - Collection en vedette
  • Collection 2 - Collection principale
  • Collection 3 - Collection secondaire
  • Collection 4 - Tracks

Configuration des pistes d'événements

Lors de la création des pistes, suivez les instructions ci-dessous pour vous assurer qu'elles apparaissent correctement dans l'expérience :

  • Les noms de collection apparaîtront dans le modèle d'événement virtuel, utilisez donc des noms significatifs
  • La collection de pistes sera une collection imbriquée, aucune vidéo n'apparaîtra dans cette collection
  • La collection de pistes devrait être la quatrième collection
  • Créer des collections enfants sous la collection de pistes pour chaque piste
  • N'utilisez pas de listes de lecture lors de la création des collections
  • La position de la section des pistes est spécifiée dans le fichier de configuration et peut être changé en éditant le breakout_collection objet dans le fichier de configuration

Configuration de l'URL des pistes

Une fois les collections créées, le fichier de configuration doit être mis à jour afin que le lien Sessions à la demande dans l'en-tête dirige vers l'URL correcte. Le nom utilisé pour la collection de pistes (collection 4) sera utilisé dans l'URL pour accéder à la page des pistes. Pour vous assurer que l'URL est correctement définie, confirmez ce qui suit.

  1. Vérifiez le fichier de configuration et localisez l' breakout_collection objet.
  2. Le collection paramètre contient l'URL de la page des pistes. Par exemple :
    
"breakout_collection": {
    "collection": "/category/videos/tracks",
    "insert_target": ".bc-on-demand-section",
    "insert_mode": "after"
},
  3. Le dernier chemin dans l' collection URL doit refléter le nom de votre collection de pistes (collection 4). Par exemple, si votre collection de morceaux (collection 4) s'appelle 'Sessions', le paramètre de collection sera mis à jour pour :
    
"collection": "/category/videos/sessions",
  4. Pour vérifier la configuration, cliquez sur le lien Sessions à la demande dans l'en-tête. La session des pistes devrait apparaître. Cliquer sur une piste devrait ouvrir les vidéos dans une piste. Si une erreur 404 s'affiche, il y a un problème de configuration.
