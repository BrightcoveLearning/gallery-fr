Personnalisation de l'en-tête d'une expérience d'événement virtuel

Dans cette rubrique, vous apprendrez comment personnaliser l'en-tête des pages Expérience d'événement virtuel.

L'en-tête apparaîtra en haut de toutes les pages. L'en-tête est construit à partir des valeurs du fichier de configuration. L'en-tête est composé des composants suivants :

  • Logo en-tête
  • Liens
  • Icônes sociales
  • Rechercher

Logo en-tête

L'image d'en-tête est spécifiée par l' images objet dans le fichier de configuration.

  • header_logo- URL de l'image devant apparaître dans l'en-tête

Les fichiers .Png et .jpg sont pris en charge. Le logo de l'en-tête sera proportionnellement réduit à 178px de large.

Liens

Par défaut, les liens HOME, LIVE STREAM et SESSIONS À LA DEMANDE sont codés en dur dans le modèle :

  • ACCUEIL - Va à la maison, haut de page
  • LIVE STREAM - Va à la collection en vedette (lecteur vidéo supérieur)
  • SESSIONS À LA DEMANDE - Aller à la section Pistes

Des liens supplémentaires peuvent être ajoutés en modifiant l' header_links objet dans le fichier de configuration.

  • link_url- URL de destination
  • label- L'étiquette doit apparaître dans l'en-tête

Icônes sociales

Par défaut, l'en-tête affichera les icônes sociales :

  • Facebook
  • Instagram
  • Twitter
  • LinkedIn
  • YouTube
Les icônes sociales qui apparaissent dans l'en-tête peuvent être modifiées/supprimées en modifiant l' social objet dans le fichier de configuration. Pour plus d'informations, voir Configuration des icônes sociales pour une expérience d'événement virtuel.

Rechercher

Pour activer la recherche et afficher le champ de recherche, activez la recherche pour l'expérience. Par défaut, le champ de recherche de l'en-tête affiche une seule ligne pour la saisie de texte.

La zone de recherche de l'exemple a été stylisée pour afficher une zone autour du champ de recherche.

Style spécifique à l'en-tête

Liens sur les appareils mobiles

Par défaut, les liens ajoutés à l'en-tête n'apparaissent pas lors de l'utilisation d'un appareil mobile. Le code JavaScript suivant résout ce problème.


for (let i = 0, l = ciabConfig.header_links.length; i < l; i++) {
  let link = ciabConfig.header_links[i];
  let li = document.createElement('li');
  let ul = document.getElementById('main-nav-cat');
  li.className = 'main-nav-category main-nav-category-home';
  li.innerHTML = ('' + link.label + ''); ul.appendChild(li);
}

Formatage du champ de recherche
