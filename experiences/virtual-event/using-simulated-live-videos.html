Utilisation de vidéos en direct simulées

{{ page.description }}

Simulive est l'abréviation de simulé en direct et c'est un moyen par lequel le contenu VOD peut être montré aux utilisateurs finaux comme s'il était en direct. Le contenu Simulive est attrayant pour les organisateurs d'événements car il permet de pré-enregistrer le contenu et d'effectuer des tests approfondis avant la mise en ligne. Il réduit les risques qui peuvent découler d'éventuelles défaillances journalières pour les événements traditionnels en direct qui se produisent en raison de problèmes d'encodeur ou d'autres aspects logistiques. Les vidéos Simulive afficheront un compte à rebours avec la vidéo pour indiquer quand il sera en direct. L'exemple ci-dessous est une vidéo simulive dans la collection en vedette.

Pour les vidéos des collections primaire et secondaire, la minuterie de compte à rebours apparaîtra sur la carte de la vidéo.

Pour utiliser du contenu simulive dans une expérience d'événement virtuel, les opérations suivantes doivent être effectuées :

  • Configurer le plugin du lecteur simulive
  • Associer le joueur à l'expérience
  • Créer un champ personnalisé simulive
  • Configurer les propriétés de la vidéo simulive
Note : Le texte qui s'affiche dans le compte à rebours se trouve dans l' labels objet du fichier de configuration et peut facilement être personnalisé ou localisé.

Configuration du plugin simulive player

Le plugin du lecteur Simulive doit être configuré pour le joueur utilisé dans votre expérience d'événement virtuel. Contactez votre gestionnaire de compte pour obtenir les informations nécessaires à la configuration du plugin.

  1. Cliquez sur Players dans l'en-tête de navigation pour ouvrir le module Players.
  2. Brightcove recommande de créer un lecteur dédié pour votre expérience d'événement virtuel. Si vous n'avez pas créé de lecteur dédié, cliquez sur Nouveau lecteur, nommez le lecteur, puis cliquez sur Enregistrer.
  3. Cliquez sur le lien du lecteur à configurer.
  4. Cliquez sur Plugins dans le menu de navigation de gauche.
  5. Cliquez sur Ajouter un plugin > personnalisé Plugin.
  6. Pour le nom du plugin, entrez simulivePlugin.
  7. Pour l'URL JavaScript et l'URL CSS, utilisez les valeurs fournies par votre responsable de compte.
  8. Cliquez sur Save pour enregistrer les modifications apportées.
Note : Toute modification apportée à un joueur peut prendre jusqu'à 5 minutes pour apparaître dans les joueurs publiés.

Configuration d'un lecteur personnalisé

Pour configurer Virtual Event Experience pour utiliser le lecteur personnalisé, procédez comme suit.

  1. Ouvrez le Galerie module.
  2. Modifiez l'expérience.
  3. Cliquez sur VIDEO AND PLAYBACK > Player et Lead Form dans la navigation de gauche.
  4. Sélectionnez l'option Utiliser un lecteur spécifique au site et sélectionnez le lecteur configuré avec le plugin Simulive.
  5. Cliquez sur Save.

Création d'un champ personnalisé simulive

Un champ personnalisé est utilisé pour stocker la date et l'heure d'une vidéo sera disponible. Pour créer le champ personnalisé simulive, procédez comme suit.

  1. Dans l'en-tête de navigation, cliquez sur ADMIN, puis sur Champs vidéo. Notez que vous devez être un administrateur de compte pour accéder au menu ADMIN.
  2. Cliquez sur Ajouter un champ personnalisé.
  3. Créez un nouveau champ personnalisé à l'aide des valeurs suivantes.
    • Nom d'affichage - Simulive Start
    • Nom interne - simulive_start
    • Type - Texte
    • Description - Le format attendu est AAAA-MM-DDTHH:MM:SS-HH:MM
  4. Cliquez sur Enregistrer le champ.
Pour plus d'informations sur la création de champs personnalisés, voir Création de champs de métadonnées personnalisés.

Configuration des propriétés vidéo simulive

Le champ personnalisé Simulive Start contiendra la date et l'heure d'une vidéo sera disponible. La vidéo ne sera pas visible tant que l'heure de début de Simulive n'aura pas été atteinte. Les vidéos Simulive doivent également être marquées simulive. Pour configurer une vidéo en tant que vidéo simulive, procédez comme suit.

  1. Ouvrez le module Médias.
  2. Cliquez sur un titre vidéo pour ouvrir les propriétés.
  3. Localisez le PROPRIÉTÉS VIDÉO rubrique et cliquez sur Éditer.
  4. Tag la vidéo avec simulive.
  5. Cliquez sur Save.
  6. Recherchez la section CHAMPS PERSONNALISÉS et cliquez sur Modifier.
  7. Saisissez une valeur de date et d'heure pour le Démarrage simulé champ. Le format correct du champ est l' YYYY-MM-DDTHH:MM:SS-HH:MM endroit où se YYYY-MM-DD trouve l'année, le mois et la date avec les zéros les plus importants, le cas échéant, HH:MM:SS est l'heure, la minute et la seconde de 24 heures avec des zéros en premier, le cas échéant, et -HH:MM représente le décalage du fuseau horaire (par exemple, - 04h00 pour UTC-4, + 08:30 pour UTC+8,5, etc.).
    Note : Simulive s'appuie sur l'heure de l'ordinateur local d'un utilisateur donné pour déterminer quand l'événement doit commencer en comparant l'heure de l'ordinateur local et l'heure de début de Simulive configurée. Pour cette raison, il peut y avoir des cas où un événement peut commencer plus tôt que prévu pour certains utilisateurs. Cela est généralement dû au fait que les heures de travail de ces' utilisateurs sont inexactes.
  8. Cliquez sur Save.

Mise à jour de la valeur clé de stratégie

Pour que le compte à rebours s'affiche sur les vidéos simulive, la valeur de la clé de stratégie dans le fichier de configuration doit être mise à jour avec une clé de stratégie activée pour la recherche. Si vous ne le faites pas, aucun compte à rebours ne sera affiché avec les vidéos. Pour obtenir une clé de stratégie activée pour la recherche, consultez Exemple de code : Générez une clé de stratégie. Une fois la nouvelle clé de stratégie créée, mettez à jour la policyKey valeur dans le fichier de configuration et republiez l'expérience.
