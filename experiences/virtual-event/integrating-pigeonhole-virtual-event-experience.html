Intégration de Pigeonhole à une expérience d'événement virtuel

{{ page.description }}

Pigeonhole peut être utilisé pour ajouter facilement une interactivité basée sur des événements (questions et réponses en direct, sondages, enquêtes, etc.) à une ou plusieurs vidéos dans une expérience d'événement virtuel. En règle générale, un nouveau Pigeonhole est créé pour l'événement en direct, puis des sessions peuvent être créées pour chaque vidéo à laquelle vous souhaitez ajouter la fonctionnalité Pigeonhole. Une fois les sessions créées, le code d'intégration iframe est associé à chaque vidéo à l'aide d'un champ personnalisé Video Cloud. Le code d'intégration entraînera l'affichage du module Pigeonhole à côté du lecteur vidéo pour les vidéos sélectionnées.

Créer un nouvel événement et une nouvelle session Pigeonhole

Vous devez créer un nouveau Pigeonhole pour votre événement en direct, puis ajouter des sessions pour chaque vidéo qui aura une fonctionnalité Pigeonhole ajoutée.

  1. Connectez-vous à votre compte Pigeonhole.
  2. Cliquez + Ajouter Pigeonhole et cliquez sur votre compte à utiliser.
  3. Entrez un nom d'événement et un code d'accès.
  4. Saisissez les dates de début et de fin de l'événement. Ces dates contrôlent combien de temps le Pigeonhole sera actif.
  5. Sélectionnez une option de fuseau horaire et de marque personnalisée (le cas échéant).
  6. Cliquez sur Continue.
  7. Maintenant que l'événement est créé, le point Agenda sera sélectionné dans la navigation de gauche. Cliquez sur Ajouter une session, puis sur Créer une nouvelle session.
  8. Sélectionner un Type de session de Questions et réponses (d'autres types de session peuvent également être utilisés).
  9. Entrez un nom de session.
  10. Entrer Début et Finir dates et heures.
  11. Configurez les paramètres Q&R selon les besoins.
  12. Cliquez sur Ajouter des questions et réponses et confirmez que la session a été créée.
  13. Créez d'autres sessions au besoin.

Génération du code d'intégration Pigeonhole

Pour qu'un Pigeonhole s'affiche à côté d'une vidéo, le code d'intégration iframe sera ajouté à la vidéo à l'aide d'un champ personnalisé Video Cloud. Le format du code sera :

<iframe src="URL du pigeonnier">< /iframe>

Pour obtenir la valeur de l'URL du pigeonhole, procédez comme suit.

  1. Dans Pigeonhole, cliquez sur Organisez votre événement dans la navigation de gauche.
  2. Cliquez sur le lien dans le Invitez vos participants section.
  3. Confirmez qu'un nouvel onglet du navigateur avec une liste de toutes les sessions s'ouvre.
  4. Cliquez sur le lien Entrée pour la session appropriée, par exemple, Saisissez cette questions-réponses. Notez que le lien va changer en fonction du type de session.
  5. Confirmez que le Pigeonhole s'ouvre dans un nouvel onglet de navigateur. L'URL Pigeonhole s'affiche dans la barre d'adresse du navigateur.

Les iframe tag avec l'URL Pigeonhole sera ajouté à la vidéo appropriée.

Configuration des paramètres d'événement

Les paramètres de l'événement Pigeonhole doivent également être configurés afin que les informations SSO puissent être utilisées pour créer un profil utilisateur.

  1. Dans la navigation de gauche, cliquez sur Plus de réglages.
  2. Clique le Participants languette.
  3. Vérifier la Autoriser la création de profils de participants via l'intégration option.
  4. Faites défiler vers le bas Profils des inscrits section sur la page.
  5. Vérifier la Activer le profil du participant pour cet événement option.

Création d'un champ personnalisé

Pour que le modèle d'événement virtuel sache quel Pigeonhole afficher pour une vidéo, un champ personnalisé est utilisé. Pour créer un champ personnalisé, procédez comme suit.

  1. Connectez-vous à Video Cloud Studio.
  2. Dans l'en-tête de navigation, cliquez sur ADMIN, puis sur Champs vidéo. Notez que vous devez être un administrateur de compte pour accéder au menu ADMIN.
  3. Créez un nouveau champ personnalisé nommé Chat Incorporer.
  4. Confirmez que le nom interne est chat_embed.
  5. Définissez le Type sur Texte.
  6. Enregistrez le champ personnalisé.

Pour plus d'informations sur la création de champs personnalisés dans Video Cloud, voir Création de champs de métadonnées personnalisés.

Ajouter un casier à une vidéo

Pour qu'un pigeonhole apparaisse en regard d'une vidéo dans le modèle Événement virtuel, procédez comme suit.

  1. Ouvrez le module Médias.
  2. Cliquez sur une vidéo pour ouvrir les propriétés de la vidéo.
  3. Recherchez la section CHAMPS PERSONNALISÉS et cliquez sur Modifier.
  4. Collez le code d'intégration de Pigeonhole dans le Intégrer le chat champ personnalisé.
  5. Cliquez sur Save.

Transmission des informations SSO de l'utilisateur à Pigeonhole

Si votre expérience d'événement virtuel est sécurisée à l'aide de SSO, il est possible de transmettre les informations SSO utilisateur à Pigeonhole afin que les utilisateurs puissent être suivis à l'intérieur de Pigeonhole. Les sections suivantes décrivent les paramètres et les étapes nécessaires pour transmettre les informations SSO à Pigeonhole.

Mapper les attributs SSO

Vous devez vous assurer que les attributs SSO corrects sont mappés afin qu'ils puissent être transmis à votre expérience, puis à Pigeonhole. L'intégration Pigeonhole recherche les valeurs suivantes :

  • fname- Nom d'utilisateur
  • nameID- Adresse e-mail de l'utilisateur

L'exemple ci-dessous montre que ces attributs sont exposés depuis la page d'administration d'Okta SSO.

Exposez les attributs dans le profil de contrôle d'accès

Assurez-vous que le profil de contrôle d'accès utilisé pour sécuriser l'expérience a le Exposez tous les attributs option cochée. Si les informations SSO ne sont pas disponibles, les utilisateurs apparaîtront comme anonymes dans Pigeonhole.

  1. Ouvrez le module Gallery.
  2. Cliquez sur Paramètres dans l'en-tête.
  3. Dans la navigation de gauche, cliquez sur Profils de contrôle d'accès.
  4. Cliquez sur l'icône de modification du profil utilisé pour sécuriser l'expérience.
  5. Assurez-vous que le Exposer tous les attributs le réglage est vérifié.

Pour plus d'informations sur la configuration de l'authentification unique, voir Configuration des profils de contrôle d'accès. Si tout est configuré correctement, les informations utilisateur s'afficheront dans Pigeonhole lorsque le spectateur se connecte à l'expérience.
