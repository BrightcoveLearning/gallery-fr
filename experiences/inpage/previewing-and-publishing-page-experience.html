Aperçu et publication d'une expérience sur la page

{{ page.description }}

Une expérience en page peut être prévisualisée à tout moment pendant la conception pour voir comment elle apparaîtra sur différents types d'appareils. La boîte de dialogue de publication permet de publier les modifications apportées à une expérience et génère également le code d'intégration que vous allez ajouter à votre page Web.

Prévisualisation d'une expérience dans la page

L'aperçu d'une expérience en page vous permet de voir comment l'expérience s'affiche sans avoir à la visualiser dans l'interface Galerie.

  1. Pour prévisualiser une expérience en page, modifiez l'expérience, puis cliquez sur Aperçu.
  2. Un aperçu du bureau s'ouvrira dans un nouvel onglet du navigateur.
    Note : Lors de l'utilisation d'une expérience en page, le style du lecteur n'est reflété que pendant la lecture d'une vidéo. Le style des joueurs ne sera pas visible dans les états BEFORE PLAY ou AFTER PLAY.
  3. Les listes déroulantes dans le coin supérieur gauche peuvent être utilisées pour prévisualiser l'expérience sur différents types de périphériques et également pendant différents états d'expérience.

Publication d'une expérience dans la page

Chaque fois que des modifications sont apportées à une expérience en page, les modifications doivent être publiées afin qu'elles apparaissent sur votre site. Si vous apportez des modifications à une expérience et que vous ne les publiez pas, les modifications n'apparaîtront pas sur le site de production.

Note : L'URL partageable et le code incorporé dans la boîte de dialogue de publication ne seront disponibles qu'après la publication de l'expérience In-Page Experience pour la première fois.
Note : Toute modification apportée à une expérience In-Page peut prendre jusqu'à 5 minutes avant d'apparaître sur les pages publiées.

Pour publier une expérience en page, procédez comme suit :

  1. Cliquez sur Publier et intégrer en haut de la page. La boîte de dialogue de publication s'ouvre.
  2. Cliquez sur Publier les modifications. Le statut de publication passe à «Publication en cours...».
  3. Une fois la publication terminée, vérifiez l' état de publication. Le statut doit être Cette expérience est à jour.  Notez que la mise à jour de l'expérience peut prendre quelques minutes.

Au fur et à mesure que des modifications sont apportées à l'expérience, vous devez republier l'expérience.

Copie du code d'intégration

Le code d'intégration est requis pour afficher l'expérience en page sur votre site. La première fois que l'expérience est publiée, le Code d'intégration peut être copié et inséré dans votre site. Au fur et à mesure que les modifications apportées à l'expérience sont apportées et publiées, ces modifications apparaîtront sur votre site.

Note : Les modifications apportées à une expérience In-Page ne nécessitent pas de remplacement de code intégré existant, SAUF si vous changez la taille de la mise en page d'une taille fixe à Responsive ou vice versa.

Il existe deux types de code intégré:

  • Recommandé - Recommandé dans la plupart des cas. L'expérience en page sera affichée à l'intérieur d'une <div> balise de votre page. Recommandé est nécessaire pour permettre à l'expérience de se redimensionner et de se comporter de manière réactive sur la page.
  • Pas de JavaScript - L'expérience In-Page sera affichée à l'intérieur d'une <iframe> balise sur votre page. Généralement, cela se fait lorsque vous intégrez l'expérience dans un CMS ou une autre plate-forme qui n' <script> autorise pas l'intégration. Il y a un risque élevé que l'expérience dans la page ne s'intègre pas toujours correctement à l'intérieur de l'intégration No JavaScript et l'expérience aura des barres de défilement ou un espace vide supplémentaire. Certaines fonctionnalités ne fonctionneront pas correctement en utilisant cette option et nous encourageons fortement les utilisateurs à intégrer à l'aide de l'intégration recommandée (JavaScript) dans la mesure du possible. Par exemple, lorsque vous utilisez cette option, analytics ne signalera pas l'URL appropriée et la lecture lightbox ne sera pas en mesure d'aller en plein écran, elle sera limitée à la taille de l'iFrame. Tout changement de hauteur de l'expérience dans les états de lecture avant/dur/après entraînera des espaces supplémentaires au bas de l'expérience ou une expérience de défilement.

Pour copier le code intégré, cliquez sur Copier le code incorporé.

Pour qu'une expérience en page soit correctement mise à l'échelle sur les appareils mobiles, il est recommandé d'ajouter une balise viewport méta au site environnant. Par exemple :

<meta name="viewport » content="width=largeur du périphérique, initial-scale=1.0" >

Pour plus d'informations sur la viewport balise, consultez ce document.

Utilisation des substitutions vidéo dynamiques

Il se peut que vous souhaitiez modifier les vidéos dans une expérience en page. Par exemple, peut-être que les vidéos d'une expérience doivent changer en fonction de la zone du site où l'expérience est intégrée. Au lieu de créer de nouvelles expériences dans la page, chacune avec un ensemble différent de vidéos, il est possible de créer une expérience, puis de modifier le code d'intégration pour attribuer dynamiquement les vidéos appropriées à une expérience. Pour plus d'informations sur l'utilisation des remplacements vidéo dynamiques, reportez-vous à la section Utilisation des substitutions vidéo dynamiques dans une expérience en page.
