    Configuration des paramètres pour une expérience dans la page

    Dans cette rubrique, vous allez apprendre à personnaliser les paramètres d'une expérience dans la page.

    Les paramètres contrôlent divers aspects d'une expérience en page. Les paramètres suivants sont disponibles :

    • Joueur
    • Vidéo
    • Social
    • SEO
    • Suivi
    • Avancé
    Remarque : Après avoir apporté des modifications à une expérience, assurez-vous de republier l'expérience afin que les modifications apparaissent sur les sites qui affichent l'expérience.

    Configuration des paramètres du lecteur

    Les paramètres du joueur contrôlent le joueur utilisé et le comportement du joueur pour l'expérience. Pour configurer les paramètres du lecteur, modifiez l'expérience, puis cliquez sur SETTINGS > Player dans la navigation de gauche.

    Sélection d'un joueur

    Pour sélectionner un joueur à utiliser pour l'expérience en page, sélectionnez un joueur dans la liste déroulante Joueurs disponibles . La liste affichera tous les lecteurs de votre compte Video Cloud.

    Après avoir sélectionné un joueur, l'aperçu du lecteur affiche un aperçu du lecteur sélectionné.

    Remarque : Lors de l'utilisation d'une expérience en page, le style du lecteur n'est reflété que pendant la lecture d'une vidéo. Le style du joueur ne sera pas visible dans les états AVANT PLAY ou AFTER PLAY.

    Définition du comportement du joueur

    Les paramètres du lecteur peuvent être configurés pour personnaliser le comportement du joueur. Les paramètres suivants sont disponibles :

    • Lorsque la lecture vidéo est terminée , détermine ce qui se passe lorsqu'une vidéo est terminée :
      • Afficher l'état « Après lecture » pour l'expérience - Lorsque cette option est sélectionnée, l'expérience entrera dans l'état AFTER PLAY lorsqu'une vidéo est terminée.
      • Rester sur la vidéo actuelle - Lorsque cette option est sélectionnée, l'expérience reste dans l'état LECTURE lorsqu'une vidéo est terminée. Sélectionnez cette option pour afficher l'écran de fin personnalisé (configuré dans le module Players) configuré pour le lecteur. Les composants affichés pendant l'état PLAY resteront également visibles. Cette option a pour effet secondaire de supprimer efficacement l'état AFTER PLAY de l'expérience.
    • Activer la lecture en ligne sur les appareils iOS : si cette option est cochée, active la lecture en ligne sur les appareils iOS lorsque vous utilisez Brightcove Player version 5.10.5 ou ultérieure.
    • Passez automatiquement à la vidéo suivante : si cette option est cochée, la vidéo suivante de l'expérience sera lue lorsque la vidéo en cours sera terminée. Ce paramètre détermine également s'il faut afficher un compte à rebours dans l'état AFTER PLAY . Si elle est activée, il y aura une minuterie. Lorsque le minuteur atteint zéro, l'expérience passe à l'état LECTURE de la vidéo suivante.

    Utilisation des comptes IP restreints

    Les comptes IP restreints en dehors de l'Amérique du Nord nécessitent une modification de l'API de lecture base_url dans le joueur. Les expériences en page copieront ceci base_url depuis le joueur lorsque l'expérience est créée et lorsqu'un joueur est sélectionné pour l'expérience. Si le base_url sur un lecteur pour une expérience existante est modifié, Gallery ne mettra pas automatiquement cette valeur à jour car nous ne savons pas qu'elle a changé. Comme solution de contournement, les éditeurs doivent suivre ces étapes comme solution de contournement :

    1. Accédez à la page Paramètres du lecteur pour l'expérience.
    2. Sélectionnez un joueur différent de celui actuellement sélectionné.
    3. Sélectionnez à nouveau le joueur initial.
    4. Enregistrez , puis publiez l'expérience.

    Cela mettra à jour le base_url pour que l'expérience corresponde à celle du joueur initial.

    Lorsque vous avez terminé, cliquez sur Enregistrer pour enregistrer les modifications.

    Utilisation des lecteurs restreints par domaine

     Les joueurs restreints au domaine nécessitent une modification de la clé de politique dans le lecteur. Expériences dans la page copiera cette policy_key à partir du joueur lorsque l'expérience est créée et lorsqu'un joueur est sélectionné pour l'expérience. Si le policy_key sur un lecteur pour une expérience existante est modifié, Gallery ne mettra pas automatiquement cette valeur à jour car nous ne savons pas qu'elle a changé. Comme solution de contournement, les éditeurs doivent suivre ces étapes comme solution de contournement s'ils mettent à jour leur liste de domaines approuvés :

    1. Accédez à la page Paramètres du lecteur pour l'expérience.
    2. Sélectionnez un joueur différent de celui actuellement sélectionné.
    3. Sélectionnez à nouveau le joueur initial.
    4. Enregistrez , puis publiez l'expérience.

    Cela mettra à jour le policy_key pour que l'expérience corresponde à celle du joueur initial.

    Lorsque vous avez terminé, cliquez sur Enregistrer pour enregistrer les modifications.

    Configuration des paramètres vidéo

    Les paramètres vidéo contrôlent certaines options d'affichage vidéo et contrôlent si la vidéo peut être téléchargée. Ces paramètres s'appliquent uniquement lorsque la vidéo est à l'état LECTURE . Pour configurer les paramètres vidéo, modifiez l'expérience, puis cliquez sur PARAMÈTRES > Vidéo dans la navigation de gauche.

    Lien connexe

    Si cette option est cochée, le lien associé à la vidéo apparaîtra avec les informations vidéo. Le lien associé est configuré dans le cadre des propriétés vidéo du module Media.

    Description

    Si cette option est cochée, la description longue de la vidéo apparaîtra au lieu de la courte description avec les informations vidéo. Les descriptions courtes et longues sont configurées dans le cadre des propriétés vidéo du module Media.

    Indicateur de visionnage

    Si cette option est cochée, un indicateur de surveillance () apparaîtra sur les vidéos qui ont été regardées pendant la session d'un utilisateur.

    Remarque : L'indicateur de surveillance sera stylé en utilisant les propriétés OVERLAY BANNER dans le menu STYLE.

    Téléchargements

    Les expériences en page peuvent être configurées pour que les spectateurs puissent télécharger des vidéos. Le lien de téléchargement permet au spectateur de télécharger le rendu de la plus haute qualité. Sélectionnez une option de téléchargement :

    • Ne pas autoriser le téléchargement de vidéos
    • Fournir un lien de téléchargement pour chaque vidéo
    • Fournissez un lien de téléchargement pour chaque vidéo en fonction d'une valeur de champ personnalisée - Définissez un champ personnalisé, puis définissez ce champ sur true pour toute vidéo pouvant être téléchargée. Pour connaître les étapes complètes de configuration d'un lien de téléchargement pour chaque vidéo, reportez-vous à la section Configuration des téléchargements vidéo pour une expérience dans la page.

    Lorsque vous avez terminé, cliquez sur Enregistrer pour enregistrer les modifications.

    Configuration des paramètres sociaux

    Les paramètres sociaux contrôlent les options de partage social disponibles lors de la lecture de vidéos. Les icônes de partage social apparaissent au-dessus du joueur lorsque l'expérience est dans l'état PLAY . Lorsqu'une vidéo est partagée sur un site de médias sociaux, un lien ou une image apparaît pour diriger le spectateur vers l'expérience.

    Gallery utilise un service proxy de métadonnées de médias sociaux. Ce service est utilisé chaque fois qu'une vidéo ou un événement est partagé à partir d'une expérience en page et fait partie de l'URL partagée. Il redirigera les spectateurs vers la page sur laquelle l'expérience en page est intégrée, mais les robots verront les métadonnées relatives à la vidéo/événement à la place. De cette façon, les robots de réseaux sociaux obtiendront des informations sur la vidéo/événement qui est partagé et peuvent présenter des informations plus pertinentes à l'utilisateur final sur le site de médias sociaux, tandis que les utilisateurs finaux peuvent toujours accéder à la page d'origine.

    Pour configurer les paramètres sociaux, modifiez l'expérience, puis cliquez sur PARAMÈTRES > Social dans la navigation de gauche.

    Remarque : Les paramètres sociaux peuvent également être configurés lors de la modification de l'expérience dans l'état LAYOUT en cliquant sur le lien Configurer le partage .

    Affichage

    Le paramètre Affichage détermine la façon dont les icônes sociales sont affichées. Lorsque l'option Partager () est sélectionnée, les icônes de partage s'affichent à partir du côté droit du lecteur.

    Lorsque l'option icônes sociales () est sélectionnée, les icônes sociales apparaîtront au-dessus du lecteur.

    Options de partage

    Sélectionnez les options de partage social à afficher. Certaines plates-formes offrent la possibilité d'entrer un message personnalisé. Cliquez sur le nom de la plateforme sociale pour entrer un message personnalisé. Cliquez sur un lien pour utiliser l'URL vidéo, le nom ou les valeurs de description courtes dans le message.

    Cliquez à nouveau sur le nom de la plate-forme sociale pour masquer les paramètres de message personnalisés.

    Métadonnées des réseaux sociaux pour les liens partagés

    Entrez l' URL de la page dans laquelle l'expérience est incorporée, puis cliquez sur Générer. Cela permettra aux sites sociaux d'afficher l'image miniature, le titre et la description de la première vidéo lorsque les spectateurs partagent votre expérience.

    Accès aux paramètres SEO

    Les paramètres SEO vous permettent de générer un plan de site et un flux de recherche pour l'expérience en page. Pour accéder aux paramètres SEO, modifiez l'expérience, puis cliquez sur PARAMÈTRES > SEO dans la navigation de gauche.

    Pour générer un plan de site et/ou un flux de recherche, procédez comme suit :

    1. Entrez l' URL de la page. Il s'agit de l'URL du site où l'expérience en page sera publiée.
    2. Vérifiez si les balises vidéo doivent être incluses.
    3. Cliquez sur Générer.

    Le code du plan du site doit être copié dans votre plan de site existant pour indexer les informations sur l'expérience en page.

    Pour configurer les paramètres d'optimisation du moteur de recherche pour une expérience de portail, modifiez l'expérience et cliquez sur Configuration du site Optimisation du moteur de > recherche dans la navigation de gauche. Les paramètres suivants sont disponibles :

    • Détails du site - Configurer l'image du site, les mots-clés et les balises méta
    • Plan du site - Accéder à la carte du site et aux URL des flux de recherche

    Configuration des détails du site

    Les paramètres Détails du site peuvent être configurés pour contrôler la façon dont le portail est indexé par les moteurs de recherche. Pour accéder aux paramètres d'optimisation du moteur de recherche, modifiez l'expérience et cliquez sur CONFIGURATION DU SITE Optimisation du moteur de > recherche dans la navigation de gauche.

    Les paramètres suivants peuvent être configurés :

    • Utiliser la description du site dans les résultats du moteur de recherche et lorsque le site est partagé - Une fois cochée, utilisera la description du site dans les balises < méta > description, y compris les balises méta Facebook et Twitter. Si cette option n'est pas cochée, la description de la vidéo en vedette est utilisée.
    • Image du site - Il s'agit de l'image qui sera affichée lorsque la page d'accueil d'un site portail est partagée sur Facebook ou Twitter.
    • Keywords : les métamots-clés qui seront ajoutés à toutes les pages du site. L'ajout de « oiseaux, animaux, poissons, monde animal » entraînera l'ajout de ce qui suit à la page HTML :
      <meta name="keywords" content="birds,animals,fish,animal world">
    • Meta Tags - Utilisé pour ajouter des balises méta supplémentaires pour le site. Ces balises seront ajoutées à toutes les pages du site. Exemple :
      <meta name="myCustomTag" content="myCustomValue">
    • Inclure des champs vidéo personnalisés dans les métadonnées de page - Lorsque cette option est cochée, ajoutera des champs personnalisés en tant que balises méta Par exemple, un champ personnalisé de l'année avec une valeur de 2018 ajoutera :
      <meta name="year" content="2018">
    • Inclure l'URL canonique dans les métadonnées de page - Une fois cochée, ajoutera une balise rel=canonical méta dans la source, ce qui aide à améliorer le référencement. Il s'agit d'une fonctionnalité qui indique à Google de traiter toutes les pages avec le même rel=canonical lien comme une seule entrée. Par exemple, mysite.com/video/monpage.html est le même que monsite.com/video/mypage. html?username=foobar.
    • Supprimer les titres vidéo de leurs URL - Lorsque cette option est cochée, le nom de la vidéo ne sera pas inclus dans l'URL de la page détaillée de la vidéo, seulement l'ID vidéo sera dans l'URL.

    Affichage de la carte du site et du flux de recherche

    Les paramètres Plan du site affichent les URL de la carte du site et du flux de recherche. Le plan peut être téléchargé et envoyé aux moteurs de recherche. Le flux de recherche peut être utilisé par la plupart des logiciels de moteur de recherche comme un moyen d'ajouter du contenu de la Galerie aux résultats de recherche. Pour accéder aux paramètres d'optimisation du moteur de recherche, modifiez l'expérience et cliquez sur CONFIGURATION DU SITE Optimisation du moteur de > recherche dans la navigation de gauche.

    Pour inclure les balises vidéo dans le plan du site, cochez l'option Inclure les balises vidéo dans le plan du site . Notez que seules les 32 premières balises sont incluses. L'ajout de balises à la carte du site les expose à des robots Web des moteurs de recherche permettant aux visiteurs de trouver des vidéos basées sur les balises vidéo.

    Remarque : ces URL ne sont pas disponibles tant que le site n'est pas publié.

    Les cartes des sites sont mises en cache pendant 24 heures. Après 24 heures, ils sont supprimés du cache et seront régénérés lorsqu'ils seront de nouveau accédés. Si des vidéos sont ajoutées à une expérience de portail, elles ne seront pas incluses dans la carte du site tant que la carte du site n'est pas régénérée (après 24 heures) ou jusqu'à ce que le site soit republié.

    Pour plus d'informations sur la création de la soumission d'un sitemap, consultez la documentation Google.

    Configuration du suivi

    Le paramètre de suivi peut être utilisé pour envoyer des données de suivi à Google Analytics. Pour configurer le paramètre de suivi, modifiez l'expérience, puis cliquez sur PARAMÈTRES > Suivi dans la navigation de gauche.

    En fournissant un ID de suivi Google Analytics sur la page Suivi, les événements survenus sur l'expérience seront soumis à un compte Google Analytics. Chaque événement a une catégorie sous la forme de Brightcove Experience {name} - {state } où {name} est le nom de l'expérience et {state} est l'état de lecture actuel du composant (avant, pendant, après). Chaque événement peut également avoir une étiquette pertinente, qui est généralement l'ID de la vidéo sur laquelle l'événement se produit. Les événements suivants font actuellement l'objet d'un suivi (d'autres seront ajoutés à l'avenir si nécessaire) :

    Action Description
    Vidéo en vedette cliqué La vidéo en vedette a été cliquée
    Vidéo cliquée Une vidéo a été cliquée
    Vidéo regardée cliqué La vidéo regardée a été cliquée
    Vidéo suivante cliquée La vidéo suivante a été cliquée
    Replay cliqué Le bouton « Replay » a été cliqué
    Arrêt d'avance automatique Le compte à rebours vidéo suivant a été arrêté/annulé
    Auto-Advance Compte à rebours écoulé, lecture de la vidéo suivante Le compte à rebours vidéo suivant s'est écoulé, de sorte que la prochaine vidéo est en cours de lecture
    Bouton Retour cliqué Le bouton « Retour » a été cliqué
    Panneau Partager ouvert Le panneau Partager a été ouvert
    Panneau de partage fermé Le panneau Partager a été fermé
    Partager sur {platform} Cliqué Le lien de partage vers la plateforme spécifiée (Facebook, Twitter, etc.) a été cliqué
     
    Remarque : Pour en savoir plus sur Google Analytics et pour créer un identifiant de suivi pour votre site, consultez le site internet de Google Analytics.

    Lorsque vous avez terminé, cliquez sur Enregistrer pour enregistrer les modifications.

    Configuration des paramètres avancés

    Les paramètres avancés peuvent être utilisés pour entrer JavaScript personnalisé ou configurer un paramètre de requête de lien vidéo. Pour configurer les paramètres personnalisés, modifiez l'expérience, puis cliquez sur PARAMÈTRES > avancés dans la navigation de gauche.

    JavaScript

    Entrez l'URL d'un fichier JavaScript personnalisé ou collez dans votre JavaScript personnalisé.

    Paramètres de requête de liaison de la vidéo

    Le paramètre de requête de lien vidéo peut être utilisé pour spécifier le paramètre de chaîne de requête qui sera utilisé lors de la liaison profonde à une vidéo. Le lien profond est généralement utilisé pour la recherche/référencement et lors du partage de vidéos sur des plateformes de médias sociaux. Par défaut, Gallery utilisera videoID.
