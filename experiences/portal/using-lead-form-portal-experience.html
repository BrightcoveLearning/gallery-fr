Utilisation d'un formulaire de prospecte avec une expérience de portail

{{ page.description }}

Les formulaires de prospect peuvent être utilisés pour capturer des informations sur les spectateurs lorsque les spectateurs regardent des vidéos et peuvent être utilisés avec les types d'expériences suivants :

  • Portail
  • Portail des événements en direct
  • Page de destination

Avec Portal, Live Event Portal et certaines expériences de Page de destination, les formulaires de prospect seront affichés à l'intérieur du joueur. Les formulaires de prospect peuvent être configurés pour s'afficher après l'affichage d'un nombre spécifié de vidéos ou après un intervalle de temps prédéterminé. Les étapes à suivre pour utiliser un formulaire de prospect sont les suivantes :

  • Créer un formulaire de prospecte à l'aide du module Audience
  • Affecter le formulaire de prospection à un lecteur compatible Audience
  • Configurer une expérience pour utiliser le formulaire de prospecte
  • Exporter les données de prospection

Création d'un formulaire de clients potentiels

Les formulaires de prospect sont créés à l'aide du module Audience. Pour connaître les étapes complètes de création d'un formulaire de prospect, y compris toutes les options disponibles, reportez-vous à la section Création d'un formulaire de prospect d'audience. Pour créer un formulaire de prospect, procédez comme suit :

  1. Ouvrez le module Audience.
  2. Cliquez sur Formulaires de prospect dans la navigation de gauche.
  3. Cliquez sur Create Lead Form.
  4. Entrez un nom pour le formulaire de prospect.
  5. Configurez les paramètres du formulaire de prospect, puis cliquez sur Enregistrer.
Note : Aucune validation n'est effectuée sur les données du formulaire de prospect saisies.
Note : Lorsqu'un formulaire de prospect est utilisé sur une expérience Landing Page et que l'expérience est configurée pour afficher le formulaire dans le corps de la page, les paramètres de synchronisation et d'affichage du formulaire de prospect sont ignorés.

Affectation d'un formulaire de prospection à un lecteur compatible Audience

Les formulaires de prospect ne peuvent être affichés que dans les lecteurs compatibles avec l'audience. Un lecteur compatible Audience est un lecteur Brightcove qui est configuré avec le plugin Audience. Les lecteurs compatibles avec le public enregistrent des données d'engagement vidéo qui peuvent être consultées à l'intérieur du module Audience. Pour créer un lecteur avec fonction Audience, procédez comme suit :

  1. Ouvrez le module Audience.
  2. Cliquez sur Lecteurs d'audience.
  3. Cliquez sur Créer un lecteur pour créer un lecteur compatible Audience.
    Note : Il est également possible d' ajouter le plugin Audience à un lecteur existant.
  4. Localisez votre joueur et utilisez la liste déroulante Formulaire de piste pour sélectionner le formulaire de prospect à afficher.
  5. (Facultatif) Si vous utilisez Audience pour suivre les données d'engagement vidéo, le lecteur doit être configuré avec une connexion de données à l'aide du Se connecter à la liste déroulante.

Configuration d'une expérience pour utiliser un formulaire de prospecte

Une fois qu'un formulaire de prospect a été créé et attribué à un lecteur compatible Audience, il peut être utilisé dans une expérience de portail. Pour configurer une expérience pour utiliser un formulaire de prospect, procédez comme suit :

  1. Ouvrez le module Gallery.
  2. Modifiez l'expérience à laquelle vous souhaitez ajouter le formulaire prospect.
  3. Cliquez sur VIDEO AND PLAYBACK > Player et Lead Form dans la navigation de gauche.
  4. Sélectionnez un lecteur compatible Audience configuré avec le formulaire prospect à afficher.
  5. Une fois qu'un joueur est sélectionné, le formulaire de prospect et les informations de connexion s'affichent. Confirmez que c'est correct.
    Note : Si une expérience Landing Page est configurée pour afficher un formulaire prospect dans le corps de la page, les paramètres de synchronisation et d'affichage du formulaire de prospect seront ignorés.
  6. Cliquez sur Save.

Assurez-vous de republier l'expérience après avoir apporté les modifications.
