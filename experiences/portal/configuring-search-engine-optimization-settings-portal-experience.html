Configuration des paramètres d'optimisation des moteurs de recherche pour une expérience de portail

{{ page.description }}

Pour configurer les paramètres d'optimisation du moteur de recherche pour une expérience de portail, modifiez l'expérience et cliquez sur Configuration du site Optimisation du moteur de > recherche dans la navigation de gauche. Les paramètres suivants sont disponibles :

  • Détails du site - Configurer l'image du site, les mots-clés et les métabalises
  • Plan du site - Accéder au plan du site et aux URL du flux de recherche

Configuration des détails du site

Les paramètres Détails du site peuvent être configurés pour contrôler la façon dont le portail est indexé par les moteurs de recherche. Pour accéder aux paramètres d'optimisation du moteur de recherche, modifiez l'expérience et cliquez sur CONFIGURATION DU SITE Optimisation du moteur de > recherche dans la navigation de gauche.

Les paramètres suivants peuvent être configurés :

  • Utiliser la description du site dans les résultats des moteurs de recherche et lorsque le site est partagé - Lorsque cette option est cochée, elle utilise la description du site dans les balises <méta> description, y compris les balises Meta Facebook et Twitter. Si cette option n'est pas cochée, la description de la vidéo en vedette est utilisée.
  • Image du site - Il s'agit de l'image qui sera affichée lorsque la page d'accueil d'un site portail est partagée sur Facebook ou Twitter.
  • Keywords : les métamots-clés qui seront ajoutés à toutes les pages du site. L'ajout de « oiseaux, animaux, poissons, monde animal » entraînera l'ajout de ce qui suit à la page HTML :
    <meta name="keywords" content="birds,animals,fish,animal world">
  • Meta Tags - Permet d'ajouter des balises méta-balises supplémentaires pour le site. Ces balises seront ajoutées à toutes les pages du site. Exemple :
    <meta name="myCustomTag" content="myCustomValue">
  • Inclure des champs vidéo personnalisés dans les métadonnées de page - Lorsque cette option est cochée, elle ajoute des champs personnalisés en tant que métabalises. Par exemple, un champ d' année personnalisé avec une valeur de 2018 ajoutera :
    <meta name="year" content="2018">
  • Inclure l'URL canonique dans les métadonnées de page - Lorsque cette option est cochée, elle ajoute une balise rel=canonical Meta dans la source, ce qui contribue à améliorer le référencement. Il s'agit d'une fonctionnalité qui indique à Google de traiter toutes les pages avec le même rel=canonical lien comme une seule entrée. Par exemple, mysite.com/video/monpage.html est identique à monsite.com/video/monpage. html?username=foobar.
  • Supprimer les titres vidéo de leurs URL - Lorsque cette option est cochée, le nom de la vidéo ne sera pas inclus dans l'URL de la page détaillée de la vidéo, seul l'identifiant vidéo sera présent dans l'URL.

Affichage de la carte du site et du flux de recherche

Les paramètres Plan du site affichent les URL de la carte du site et du flux de recherche. Le plan peut être téléchargé et envoyé aux moteurs de recherche. Le flux de recherche peut être utilisé par la plupart des logiciels de moteur de recherche comme un moyen d'ajouter du contenu de la Galerie aux résultats de recherche. Pour accéder aux paramètres d'optimisation du moteur de recherche, modifiez l'expérience et cliquez sur CONFIGURATION DU SITE Optimisation du moteur de > recherche dans la navigation de gauche.

Pour inclure les balises vidéo dans le plan du site, cochez la case Inclure des balises vidéo dans le plan du site option. Notez que seules les 32 premières balises sont incluses. L'ajout de balises à la carte du site les expose à des robots Web des moteurs de recherche permettant aux visiteurs de trouver des vidéos basées sur les balises vidéo.

Note : Ces URL ne seront pas disponibles tant que le site n'aura pas été publié.

Les cartes des sites sont mises en cache pendant 24 heures. Après 24 heures, ils sont supprimés du cache et seront régénérés lorsqu'ils seront de nouveau accédés. Si des vidéos sont ajoutées à une expérience de portail, elles ne seront pas incluses dans la carte du site tant que la carte du site n'est pas régénérée (après 24 heures) ou jusqu'à ce que le site soit republié.

Pour plus d'informations sur la création de la soumission d'un sitemap, consultez la documentation Google.
