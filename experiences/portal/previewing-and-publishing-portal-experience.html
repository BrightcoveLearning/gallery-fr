Aperçu et publication d'une expérience de portail

{{ page.description }}

Les expériences de portail peuvent facilement être prévisualisées pendant le développement. Vous pouvez également partager des liens vers l'aperçu du site. Une fois le site terminé, vous pouvez le publier pour le grand public.

Prévisualisation d'une expérience de portail

L'aperçu d'une expérience vous permet de voir comment l'expérience va s'afficher sur les appareils mobiles, tablettes et ordinateurs de bureau. Pour prévisualiser une expérience, modifiez l'expérience, puis cliquez sur Aperçu dans l'éditeur de site Galerie.

Il y a deux options :

  • Aperçu réactif - Ouvre l'expérience dans une nouvelle fenêtre de navigateur pour un aperçu
  • Get Shareable Link - Fournit une URL de prévisualisation afin que l'aperçu puisse être partagé avec d'autres personnes.

L'URL partageable générée est valide pendant 72 heures.

Note : Si vous avez configuré un délai d'expiration de session dans le cadre de tous les paramètres de contrôle d'accès, le délai d'attente de 72 heures remplacera le délai d'expiration de session configuré. Lorsque l'expérience est publiée, le délai d'expiration de la session (le cas échéant) sera appliqué.

Publication d'une expérience de portail

La publication d'une expérience de portail la rend accessible aux utilisateurs et génère une URL statique pour le site. Pour publier une expérience, procédez comme suit :

Note : Si vous avez récemment apporté des modifications aux métadonnées vidéo ou aux listes de lecture, ces modifications peuvent prendre jusqu'à 6 minutes avant d'apparaître sur les expériences publiées.
  1. Cliquez sur le nom de l'expérience pour ouvrir le site dans l'éditeur de site.
  2. Cliquez sur Publish.
  3. Dans la boîte de dialogue Publish Confirmation, cliquez sur Publish Now.
  4. L'expérience sera publiée et l'URL du site sera affichée.

Le bcvp0rtal domaine prend en charge SSL par défaut, de sorte que les sites peuvent également être consultés à l'aide de https ://, aucun certificat SSL ou configuration n'est requis. Il est également possible de sécuriser le site en utilisant vos propres certificats SSL. Pour plus d'informations, consultez Sécurisation d'une expérience de portail avec SSL.

Note : Pour plus d'informations sur la façon d'héberger l'expérience sur votre propre domaine, consultez Affectation d'un domaine personnalisé à une expérience de portail.

Une fois l'expérience publiée, l'URL sera affichée dans les détails du site sur la page d'accueil de la Galerie.

Publication des modifications apportées à une expérience de portail

Lorsque vous apportez des modifications à une expérience, vous devez republier le site pour les rendre disponibles. Les expériences qui doivent être republiées afficheront un statut d'expérience des modifications non publiées dans les détails du site.

Pour republier les modifications, suivez les étapes décrites dans la section Publication d'un site de cette rubrique.

Note : Il se peut que vous deviez vider le cache du navigateur et les cookies si les modifications n'apparaissent pas.

Obtenir une expérience de portail indexée par les moteurs de recherche

Une fois le site publié, il y a quelques étapes que vous pouvez prendre pour faciliter l'indexation du site par les moteurs de recherche. Pour plus d'informations sur cette rubrique, voir Indexation d'une expérience de portail Gallery.

Mise hors ligne d'une expérience de portail

En prenant une expérience hors ligne, elle n'est plus disponible à son URL. Pour mettre une expérience hors ligne, cliquez sur une ligne, puis cliquez sur Prendre hors ligne. Pour confirmer que l'expérience doit être mise hors ligne, cliquez sur Déconnexion. Pour que l'expérience soit à nouveau disponible, l'expérience doit être republiée. Une fois l'expérience hors ligne, elle ne sera plus accessible via son URL.

Note : Le fait de mettre l'expérience hors ligne ne supprime pas l'expérience.
