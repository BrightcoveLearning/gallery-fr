Ajout de questions-réponses Pigeonhole à une expérience de portail d'événements en direct

{{ page.description }}

Pigeonhole peut être utilisé pour ajouter facilement une interactivité basée sur des événements (questions et réponses en direct, sondages, enquêtes, etc.) à une expérience de portail d'événements en direct. Généralement, un nouveau Pigeonhole est créé pour l'événement, puis des sessions sont créées pour les événements en direct. Une fois la session créée, le code d'intégration Pigeonhole est collé dans le modèle Live Event. Le code d'intégration entraînera l'affichage du pod Q&A Pigeonhole à côté du joueur de l'événement en direct.

Création d'un événement et d'une session Pigeonhole

Vous devez créer un nouveau Pigeonhole pour votre événement en direct, puis ajouter des sessions pour chaque vidéo qui aura Q&R activé.

  1. Connectez-vous à votre compte Pigeonhole.
  2. Cliquez sur + Ajouter Pigeonhole et cliquez sur le compte à utiliser.
  3. Entrez un nom d'événement et un code d'accès.
  4. Saisissez les dates de début et de fin de l'événement. Ces dates contrôlent combien de temps le Pigeonhole sera actif.
  5. Sélectionnez une option de fuseau horaire et de marque personnalisée (le cas échéant).
  6. Cliquez sur Continue.
  7. Maintenant que l'événement est créé, le point Agenda sera sélectionné dans la navigation de gauche. Cliquez sur Ajouter > une session Créer un nouveau.
  8. Sélectionnez un type de session de questions-réponses.
  9. Entrez un nom de session.
  10. Entrer Début et Finir dates et heures.
  11. Configurez les paramètres Q&R selon les besoins.
  12. Cliquez sur Ajouter des questions et réponses. Confirmez que la session a été créée.
  13. Cliquez sur Plus de paramètres dans la navigation de gauche.
  14. Cliquez sur l'onglet Intégrations en haut de la page.
  15. Faites défiler jusqu'au Autres section.
  16. Recherchez Brightcove Gallery Incorporer et cliquez sur Intégrer.
  17. (Facultatif) Si votre expérience Gallery est sécurisée à l'aide de l'authentification unique, sélectionnez Autoriser la connexion automatique des participants à l'aide des attributs SAML. Cela récupère le nom et l'adresse e-mail de l'utilisateur et les connectera automatiquement sur Pigeonhole Live lorsque le widget Pigeonhole Live se charge. Si cette option n'est pas cochée, les utilisateurs apparaîtront comme anonymes à l'intérieur de Pigeonhole. Cliquez sur Save.
  18. Dans la boîte de dialogue Incorporer Brightcove Gallery, sélectionnez votre session Q&R dans la liste déroulante.
  19. Cliquez sur Obtenir le code intégré.
  20. Cliquez sur le code d'intégration et copiez-le dans le presse-papiers. Il s'agit du code qui sera intégré à l'expérience Galerie.
  21. Cliquez sur Close (Fermer).

Ajout d'un pigeonhole à une expérience

Le code d'intégration Pigeonhole doit être ajouté à une expérience pour qu'un Pigeonhole puisse s'afficher. Pour ajouter le code d'intégration à une expérience, procédez comme suit.

  1. Ouvrez le module Gallery.
  2. Cliquez sur le Live Event Experience pour modifier.
  3. Clique sur le Paramètres Twitter relier.
  4. Dans le Fil Twitter section, collez le code d'intégration Pigeonhole.
  5. (Optionnel) Collez le code d'intégration dans le Habitent et Après l'occassion pages aussi.
  6. Enregistrez et publiez les modifications.

Passer des valeurs SSO à la Galerie

Si votre expérience Galerie est sécurisée à l'aide de l'authentification unique et que vous avez sélectionné l'option Autoriser la connexion automatique des participants à l'aide des attributs SAML lors de la création de votre session, plusieurs choses doivent être effectuées pour garantir que les attributs SSO sont disponibles pour Pigeonhole. Si les informations SSO ne sont pas disponibles, les utilisateurs apparaîtront comme anonymes dans Pigeonhole.

  1. Ouvrez le module Gallery.
  2. Cliquez sur Paramètres dans l'en-tête.
  3. Dans la navigation de gauche, cliquez sur Profils de contrôle d'accès.
  4. Cliquez sur l'icône de modification du profil utilisé pour sécuriser l'expérience.
  5. Assurez-vous que le Exposer tous les attributs le réglage est vérifié.

Vous devez également vous assurer que les attributs SSO corrects sont mappés afin qu'ils soient accessibles par Pigeonhole. Pigeonhole recherche les valeurs suivantes :

  • emailaddress
  • name

L'exemple ci-dessous montre que ces attributs sont exposés à partir de la page Okta SSOAdmin.

Si tout est configuré correctement, les informations utilisateur s'afficheront dans Pigeonhole lorsque le spectateur se connecte à l'expérience.
